Es para realizar test pero sin la necesidad de usar el sistema real

Se puede usar una clase que tabaje por Ej como una DB, entonces se le crea los metodos para que esta clase responda como una DB


Patrones (Cascarón vacío → Simulación)
• Test Stub: cascarón vacío. Sirve para que el SUT envie los mensajes esperados
• Test Spy: Test Stub + registro de outputs
• Mock Object: test Stub + verification of outputs
• Fake Object: imitación. Se comporta como el módulo real (protocolos, tiempos de respuesta, etc)
